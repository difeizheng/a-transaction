# 实时行情数据源增强 - 新浪财经/腾讯财经接入

**实施日期**: 2026-03-25
**实施状态**: ✅ 完成
**测试验证**: ✅ 通过

---

## 一、背景

之前系统实时行情依赖 AkShare，但经常出现 `RemoteDisconnected` 错误。根据评估：

| 数据源 | 响应时间 | 稳定性 | 费用 |
|--------|----------|--------|------|
| 新浪财经 | 100-300ms | ★★★★★ | 免费 |
| 腾讯财经 | 150-400ms | ★★★★☆ | 免费 |
| 东方财富 | 200-500ms | ★★★★☆ | 免费 |
| AkShare | 500-2000ms | ★★★☆☆ | 免费 |
| Tushare | 300-800ms | ★★★★☆ | 部分收费 |

**新浪财经和腾讯财经是最稳定、最快的免费实时行情数据源。**

---

## 二、实施方案

### 2.1 数据源优先级（实时行情）

```
新浪财经 > 腾讯财经 > AkShare > 东方财富 API
```

### 2.2 新增接口

#### `_get_from_sina()` - 新浪财经（优先级最高）
- **接口**: `http://hq.sinajs.cn/list=[市场前缀][股票代码]`
- **响应时间**: 100-300ms
- **数据格式**: `var hq_str_sh000001="名称，开盘，昨收，当前，最高，最低，..."`
- **优点**: 最稳定、最快、数据最全

#### `_get_from_tencent()` - 腾讯财经（备用）
- **接口**: `http://qt.gtimg.cn/[市场前缀][股票代码]`
- **响应时间**: 150-400ms
- **数据格式**: `v_sh600000="51~浦发银行~600000~7.55~7.54~7.56..."`
- **优点**: 数据丰富，支持更多字段

#### `_get_from_em_api()` - 东方财富（降级）
- **接口**: `http://push2.eastmoney.com/api/qt/stock/get`
- **响应时间**: 200-500ms
- **优点**: 兜底方案
